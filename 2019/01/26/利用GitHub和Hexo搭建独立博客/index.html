前言

这是一篇纯小白篇的搭建博客的博文，也是我搭建好这个博客发布出的第一篇文章。在我搭建这个独立博客的时候遇到了许多大大小小的问题，有一部分通过别人的教程一步一步的解决了，有些在搜索引擎上也得到了一定的答案，但并不是所有答案都得到了解决，所以我打算我的第一篇博客就记录下我搭建的全过程吧 正如我开头时就已表明的那样，也正好复习和巩固。

Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

一. Hexo的安装

桌面右击鼠标，点击Git Bash Here依次输入npm命令安装

npm install hexo-cli -g

npm install –save

#如果npm命令无法正常运行，或者报错的话就用淘宝的npm源(cnpm)#

在Git Bash Here里执行 npm install -g cnpm –registry=https://registry.npm.taobao.org 因为这里将npm换成了淘宝的cnpm，所以以后的npm命令统一改成cnpm来执行

二. Hexo的初始化设置（建立Hexo相关文件夹）

创建Hexo文件夹，任何一个盘都可以但最好别在C盘，例如我创建的路径是F:\Blog\Hexo

在Hexo文件夹下右键打开Git Bash Here在里面依次运行如下命令

hexo init

npm install

分清自己换没换npm源，这里我是用npm做例子的

运行完上面的命令后Hexo会在你指定的文件夹下（也就是F:\Blog\Hexo）生成所需要的文件，生成后的Hexo目录如下：

不一定完全一致，以自己生成的为准只要接下来需要找的文件都有就没问题

三. 配置SSH密钥

配置Github的SSH密钥可以让本地git项目与远程的github建立联系，让我们在本地写了代码之后直接通过git操作就可以实现本地代码库与Github代码库同步。

操作如下：

  1. 查看自己本机是否存在SSH密钥(keys)，在桌面上打开Git Bash Here输入并运行

    cd ~/. ssh

如果执行完显示No such file or directory则说明此电脑上没有SSH密钥。

  • 如果没有的话需要我们自己生成新的SSH Key：

    输入ssh-keygen -t rsa -C “邮件地址@youremail.com”后会生成

    Generating public/private rsa key pair.

Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>

注意：

此处的邮箱地址输自己的，此处的[-C]是大写的[C].

随后系统会让你输入密码：

Enter passphrase (empty for no passphrase):<输入加密串>

Enter same passphrase again:<再次输入加密串>

在回车的时候会提示让你输入一个密码，这个密码会在提交项目的时候使用。这个设置是防止别人往自己的项目里提交内容。

注意：在输入密码的时候并没有*号等标识，尽管输入完enter就好。

设置成功后会看到如下的画面：

现在说明我们的SSH Key已经设置好了。

添加 SSH key 到GitHub上

在本地上设置好了SSH后需要添加到GitHub上，以此来完成本地到远程服务器的连接。

  • 查看自己的SSH Key

有两种方法

1. 在桌面上右键打开Git Bash Here,执行cd ~/.ssh这个命令
2. 打开目录文件C:\Users\user.ssh\id_rsa.pub查看
  • 在GitHub账户中添加SSH key公钥

1.复制自己的SSH，登录GitHub进入Account Settings

2.选择SSH keys

3.粘贴密钥，添加即可

  • 测试

输入下面的命令，查看是否设置成功，git@github.com部分不用修改

ssh -T git@github.com

如果是下面的反馈：

The authenticity of host 'github.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

直接输入yes就好，然后会看到：

Hi yourname! You've successfully authenticated, but GitHub does not provide shell access.

其中如果之前在添加SSH时设了密码的话，仔细看显示的内容 有些步骤是需要输入你当时设定的密码才能正常进行的，用心观察就好

  • 设置用户信息

现在已经可以通过本地SSH连接到远程的GitHub了，但是有些个人信息需要完善，Git会根据用户的名字和邮箱来进行提交。GitHub也是用这些信息来做权限处理的，输入下面的代码进行个人信息的设置，把名称和邮箱换成自己的。

git config --global user.name "lnitial-heart"//自己用户名
git config --global user.email "2302623337@qq.com"//自己的邮箱
  • SSH Key配置成功

本机已经成功连接到GitHub了。下面是一些问题官方给出的解决文档：
